## High-level architecture

```
┌─────────────────────────────────────────────────┐
│                  miga binary                    │
│                                                 │
│  main.rs  ──►  cli.rs (clap)                    │
│                   │                             │
│          ┌────────┴─────────────────────────┐   │
│          │        commands/                 │   │
│          │  init · add · fetch · run ·      │   │
│          │  build · remove                  │   │
│          └────┬───────────────┬─────────────┘   │
│               │               │                 │
│         compiler/        registry/              │
│         (oxc pipeline)   (manifest + HTTP)      │
│               │               │                 │
│          utils/ ─────────────────────────────── │
│          fs · json · npm · env · builder ·      │
│          output · tsconfig · net · project      │
└─────────────────────────────────────────────────┘
```

All commands return `anyhow::Result<()>`. Errors bubble up to `main.rs` where a
single `output::error()` call formats and exits.

## Binary entry point

**`src/main.rs`**

1. Parses the CLI with `Cli::parse()` (clap derive).
2. Matches on `Commands` and delegates to the appropriate `commands::*::run()`.
3. On error: prints a formatted message via `utils::output::error()` and exits
   with code 1.

---

## CLI layer

**`src/cli.rs`**

Defines `Cli` and `Commands` using clap's derive macros.

| Subcommand | Struct variant                       |
| ---------- | ------------------------------------ |
| `init`     | `Commands::Init { namespace, name }` |
| `add`      | `Commands::Add { packages }`         |
| `fetch`    | `Commands::Fetch { modules }`        |
| `run`      | `Commands::Run { obfuscate }`        |
| `build`    | `Commands::Build { obfuscate }`      |
| `remove`   | `Commands::Remove { module }`        |

## Commands

### `init` (`src/commands/init.rs`)

1. Collects metadata interactively via `dialoguer::Input`.
2. Validates namespace (lowercase, no spaces/colons) and name.
3. Creates the project directory and sets it as the working directory.
4. Generates fresh UUIDs for all manifest entries.
5. Writes BP and RP directory trees, manifests, entry-point scripts,
   license files and static templates.
6. Writes `.miga/miga.json` (project manifest) and `.miga/modules.lock`.
7. Calls `commands::add::run()` to install the chosen `@minecraft/server` version.

**Static content** is defined as `const` strings near the bottom of the file.
**Dynamic content** (manifests, licenses, README) is generated by dedicated
`fn` helpers that accept only what they need.

---

### `add` (`src/commands/add.rs`)

Downloads TypeScript type packages from the npm registry.

1. `utils::net::is_online()` — connectivity check.
2. `utils::project::require_initialized()` — ensures `.miga/` exists.
3. For each package spec: calls `utils::npm::fetch_types()`.
4. Records `name → version` in `.miga/miga.json` under `externals`.
5. Updates `behavior/manifest.json` dependencies block via
   `sync_behavior_manifest()`.

---

### `fetch` (`src/commands/fetch.rs`)

Installs modules from the miga registry.

1. Connectivity check.
2. `utils::project::require_initialized()`.
3. For each module name: resolves full dependency tree via
   `registry::resolve_dependencies()`.
4. Downloads the `.tar.gz` archive and extracts it.
5. Records the module in `.miga/modules.lock`.

---

### `run` (`src/commands/run.rs`)

Hot-reload watch loop.

1. `utils::builder::build_project()` — initial compile and deploy.
2. `notify::RecommendedWatcher` watches `behavior/scripts/`.
3. On any change event: re-runs `build_project()`.

---

### `build` (`src/commands/build.rs`)

Full compile and package.

1. `utils::builder::build_project()` — TypeScript → JS compile + deploy.
2. Syncs version from `.miga/miga.json` into both manifests.
3. `dist/` is cleaned, then `zip` archives the BP and RP into `.mcpack` files
   and combines both into a `.mcaddon`.

---

### `remove` (`src/commands/remove.rs`)

Removes a registry module.

1. `utils::project::require_initialized()`.
2. Reads lock file; verifies the module is installed.
3. Removes the module's files from disk (`behavior/scripts/` and
   `.miga_modules/`).
4. Removes the dependency edge from `behavior/manifest.json`.
5. Writes the updated lock file.

## Compiler pipeline

**`src/compiler/mod.rs`**

Uses the [oxc](https://oxc.rs/) family of crates for zero-Node.js TypeScript
compilation.

```
.ts source
    │
    ▼
oxc_parser::Parser::parse()          — produces AST
    │
    ▼
oxc_semantic::SemanticBuilder        — scope & binding analysis, produces
                                       the Scoping passed to the Transformer
    │
    ▼
oxc_transformer::Transformer         — TypeScript → ES2020 JS
    │
    ▼
(optional) oxc_minifier::Minifier    — dead-code elimination + mangling
    │
    ▼
oxc_codegen::Codegen                 — emit final JS string
```

**Import rewriting**: relative `./` and `../` imports have their `.ts`
extension replaced by `.js`. Bare `minecraft:` specifiers are left as-is.
All other bare specifiers are prefixed with the computed relative path to
`.miga_modules/`.

## Registry protocol

**`src/registry/mod.rs`**

The registry base URL is read from `MIGA_REGISTRY_URL` (falls back to a
hard-coded default). Two HTTP endpoints are used:

| Endpoint                            | Purpose                                       |
| ----------------------------------- | --------------------------------------------- |
| `GET /registry.json`                | Global registry manifest listing all modules. |
| `GET /modules/<name>/<name>.tar.gz` | Module archive download.                      |

**`src/registry/manifest.rs`** defines `ProjectManifest` (`.miga/miga.json`)
and `ModuleManifest` (per-module descriptor inside archives).

## Shared utilities

| Module                                | Key responsibilities                                                                                                        |
| ------------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| `utils/project.rs`                    | `require_initialized`, `load_manifest`, `save_manifest`, `load_lock`, `save_lock` — single source of truth for project I/O. |
| `utils/net.rs`                        | `is_online()` — TCP probe to `1.1.1.1:53` with a 1500 ms timeout.                                                           |
| `utils/builder.rs`                    | `build_project(opts)` — orchestrates compile → copy to deploy paths.                                                        |
| `utils/compiler/` → `compiler/mod.rs` | oxc-based TypeScript → JS pipeline.                                                                                         |
| `utils/env.rs`                        | Resolves `DeployPaths` from `.env` or platform defaults.                                                                    |
| `utils/fs.rs`                         | `ensure_dir`, `clean_dir`, `copy_dir`, `exists`, `write_if_not_exists`.                                                     |
| `utils/json.rs`                       | `read_json`, `write_json`, `to_unicode_escapes`.                                                                            |
| `utils/npm.rs`                        | Downloads `.d.ts` files from the npm registry.                                                                              |
| `utils/output.rs`                     | Consistent terminal output helpers (`section`, `step`, `success`, `error`, `warn`).                                         |
| `utils/tsconfig.rs`                   | Writes the TypeScript path-mapping `tsconfig.json` for module resolution.                                                   |

## Key data structures

### `ProjectManifest` (`.miga/miga.json`)

```json
{
    "name": "my-addon",
    "namespace": "woc",
    "version": "0.1.0",
    "modules": { "<module-name>": "<version>" },
    "externals": { "@minecraft/server": "2.4.0" }
}
```

### `LockFile` (`.miga/modules.lock`)

```json
{
    "modules": {
        "<module-name>": {
            "version": "1.0.0",
            "files": ["behavior/scripts/...", "..."]
        }
    }
}
```

### `ModuleManifest` (inside each module archive)

```json
{
    "name": "module-name",
    "version": "1.0.0",
    "description": "...",
    "dependencies": ["other-module"]
}
```

### `CompileOptions`

```rust
pub struct CompileOptions {
    pub obfuscate: bool,
    pub source_maps: bool,
}
```
